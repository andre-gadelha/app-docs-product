Criar a estrutura de um APP baseado no FrameWork Flask compatível com a versão Python 3.11.9 onde:

	O sistema deve possuir uma tela de autenticação com única opção para realização de login usando oAuth do Google.

	Somente após a autenticação no google, o usuário poderá acessar a tela principal que deve conter os seguintes requisitos:

		O sistema deve possuir uma tela principal no padrão MasterpPage(onde todas as páginas filhas devem ser acopladas ao centro da pagina principal quando acionadas pelo menu/submenus de navegação) com os Requisitos:

			 Deve apresentar o nome do usuário no canto superior direito da tela principal com opção para logoff.

			 Deve conter um Menu Principal com as seguintes opções: Documentos do Projeto e Documentos de Requisitos:
			
				Como submenus de Documentos do Projeto, devem existir as seguintes opções: Proposta de OS e Relatório de Entrega.
				
					-Ao acionar a opção Proposta de OS:
					
						O sistema deve abrir uma página contendo um formulário com os campos de acordo com as seções disponíveis no arquivo template 'Proposta OS 'AnoOS'-'NúmeroOs' - 'TítuloOS'.docx', onde: 
					
							O arquivo deve: 
								
								Ser criado em formato docx com a seguinte estrutura no nome: Proposta OS 'AnoOS'-'NúmeroOs' - 'TítuloOS'.docx, onde: 
								
									Para o nome do documento: 
									
										Os termos entre '' devem ser criados os campos para serem informados pelos usuários no formulário do Front End.
										
									Para o conteúdo do documento:
									
										Os termos entre <> devem ser criados os campos para serem informados pelos usuários no formulário do Front End.
										
											<Nome da Os> deve ser informado em um único campo e preenchido automaticamente nos campos referentes do conteúdo do socumento.
											<Tipo da OS> deve ser informado em um único campo e preenchido automaticamente nos campos referentes do conteúdo do socumento.
											<Quantidade de HST> deve ser informado em um único campo e preenchido automaticamente nos campos referentes do conteúdo do socumento.
											<Itens da OS> deve permitir que o usuário informe vários Itens e cada ves que o usuário informar um Item e acionar a opção +(adicionar) o sistema deve numerar no formulário os itens que serão inseridos no documento. Permirindo que o usuário edite ou exclua itens no formulário reordenando a numeração dos itens quando houve exclusões.
											<Data atual do servidor> deve ser informado em um único campo e preenchido automaticamente nos campos referentes do conteúdo do socumento.
										
											Campos para que o sistema faça o preenchimento automático:
											
												<Data atual do servidor>;
												<Cálculo em R$ do valor de HST x 200,00> - Conforme a quantidade de HSTs
										
											Listas previstas para os campos no formulário:
											
												Tipo da OS: Concepção, Execução e Treinamento
						
						No formulário deve ter um botão para acionar a geração do documento onde o sistema deve gerar fielmente as formatação do template informado para gerar o documento;
						No formulário deve ter um botão para limpar o formulário.
					
					-Ao acionar a opção Relatório de Entrega:
					
						O sistema deve abrir uma página informando que a funcionalidade esta em construção.
